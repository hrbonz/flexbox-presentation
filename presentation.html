FlexWhat?

Flexbox is a layout model for CSS3, it allows the browser to arrange elements on the screen in a predictive way.

Flexbox is fluid, responsive, adaptive

Should I use flexbox?

Short answer is: YES

(except if you have to support IE8/9)

There are still some quirks and bugs and support is not complete across the board (caniuse.com) but most of it is working fine on most platforms/browsers

Main issue: wrapping elements

Let's dive in - basic concepts

There are 2 main elements in a flexbox layout:

  • Containers
  • Items

Let's dive in - basic concepts

A container is used to wrap your flex items

It defines the flow, distribution and alignment of the items

Let's dive in - basic concepts

An item is an element in the flexbox flow

It can alter its size, relative size, order and alignment

Here be dragons


.mycontainer {
    display: flex;
}

Let see a basic example

Important: only the direct children elements are affected

Here be dragons

for better compatibility:


.mycontainer {
    display: -ms-flex;
    display: -webkit-flex;
    display: flex;
}

The -webkit- and -ms- prefixes are ensuring the extra.

Now, the first magic trick

Behold the almighty absolute centering


html,
body {
    height: 100%;
}

.container {
    display: flex;
    height: 100%;
}

.item {
    margin: auto;
}

Flow direction


.container {
    display: flex;
    flex-direction: column;
}

Options are: row (default), column, row-reverse, column-reverse

Example

Align items


.container {
    align-items: flex-start;
    display: flex;
}

Options are: stretch (default), flex-start, flex-end, center, baseline

Example

Justify content


.container {
    display: flex;
    justify-content: flex-end;
}

Options are: flex-start (default), flex-end, center, space-between, space-around

Example

Items order


.item {
    flex: 1;
}

.item-1 {
    order: 2;
}

.item-2 {
    order: 3;
}

.item-3 {
    order: 1;
}

Example

Items distribution


.item-1 {
    flex: 3;
}

.item-2,
.item-3 {
